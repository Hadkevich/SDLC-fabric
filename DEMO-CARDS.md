# 🧠 КАРТОЧКА 1 · Решение принимал агент-архитектор
### ADR-003 — Бюджет задержки 500 мс: входит ли в него вызов LLM?

**Противоречие в требованиях:** ответ `/matches` должен прийти за **< 500 мс** И содержать объяснение — но вызов LLM сам по себе ~**1200 мс**.

| Вариант | Вердикт |
|---|---|
| Всё синхронно, включая вызов LLM | ❌ невозможно: LLM ~1200 мс |
| Только скоринг за 500 мс, объяснение — поллингом | ⚠️ клиенту нужен polling |
| **Скоринг + детерминированная заглушка за 500 мс; полное LLM-объяснение из кэша / async** | ✅ **ВЫБРАН** |

**Почему так:**
- Ответ всегда **< 500 мс** (§8 NFR), объяснение не пустое: готовая заглушка **или** кэш-LLM
- Заглушка детерминированно собрана из скоров (навыки / стиль / рост) — **это то, что вы видите в UI**
- Ключ кэша включает хэш весов → смена весов корректно сбрасывает кэш *(это бит 6!)*

> 💬 Агент не «выбирал библиотеку» — он разрешил **реальное противоречие в самих требованиях** про латентность. Аргументированное, проверяемое инженерное решение.

# 🛡️ КАРТОЧКА 2 · Система поймала и исправила свой баг
### BLK-001 — несущий гейт в действии

**BEFORE — reviewer поймал дефект**
> `GET /teams/{team_id}/risk-summary` → **HTTP 501** «не реализовано»
> Reviewer (на более сильной модели, чем developer) пометил это как блокирующее.

**Event log — гейт держал линию:**
```
deployment · devops-agent · BLOCKED   ← отказ выкатывать
deployment · devops-agent · BLOCKED
deployment · devops-agent · BLOCKED
   … developer чинит, reviewer переодобряет …
deployment · devops-agent · SUCCESS   ← только теперь деплой
```

**AFTER — доведено до ума**
> Тот же эндпоинт → **HTTP 200**, отдаёт `TeamRiskSummary`
> → это **живой риск-дашборд менеджера**, который покажем в бите 6.

> 💬 Автономная самокоррекция: ни один человек не чинил баг руками — система поймала свой дефект и не пустила его в прод.
